TeamChat - リアルタイムチームコミュニケーションプラットフォーム

■ 概要

TeamChatは、日本の中小企業（従業員50〜500人）向けに設計された
チームコミュニケーションツールです。
Slackライクな機能を日本語UIと国内データセンター（AWS東京リージョン）で提供し、
セキュリティとコンプライアンスを重視する企業ニーズに応えます。

■ 解決する課題

1. 海外サービスへのデータ移転に対する不安（個人情報保護法改正2025年施行）
2. 既存ツール（メール、FAX）からの移行の難しさ
3. Microsoft Teamsの「Teams疲れ」（通知過多、UIの複雑さ）
4. Slackの高コスト（月額925円は中小企業に負担）
5. 金融・公共セクターのISMAP対応要件

■ 主な機能

基本機能:
- チャンネルベースのメッセージング（テキスト、画像、ファイル共有）
- スレッド機能によるトピック別議論
- ビデオ通話・画面共有（最大50人）
- タスク管理（Chatworkの強みを取り込み）
- 高度な検索機能（ユーザーの最大要望に対応）

差別化機能:
- AIアシスタント（議事録自動生成、タスク抽出、要約）
- 監査ログ・情報漏洩防止（DLP）機能
- オンプレミス版の提供（金融・公共向け）
- ISMAP対応（政府・自治体利用要件を満たす）
- 既読・未読管理（ユーザー要望24.2%に対応）

外部連携:
- Google Workspace連携
- Microsoft 365連携
- kintone（サイボウズ）連携
- Backlog（ヌーラボ）連携
- freee/マネーフォワード（会計ソフト）連携
- Salesforce連携
- 初期リリース時点で50+の連携（Slackの2,600+には及ばないがChatworkの600+と同等を目標）

■ 技術スタック

- フロントエンド: React + TypeScript
- バックエンド: Go + gRPC
- リアルタイム通信: WebSocket
- データベース: PostgreSQL + Redis
- 検索エンジン: Elasticsearch
- AI基盤: Claude API（議事録・要約）
- インフラ: AWS東京リージョン（オンプレミス版はKubernetes対応）
- セキュリティ: E2E暗号化、ゼロトラスト対応

■ 価格プラン

- フリープラン: 10人まで無料、メッセージ履歴1万件、連携3つまで
- スタンダード: 月額500円/ユーザー、無制限履歴、外部連携無制限
- ビジネス: 月額1,200円/ユーザー、SSO、監査ログ、DLP、SLA99.9%
- エンタープライズ: 要見積、オンプレミス、専用サポート、ISMAP対応

価格ポジショニング（競合比較）:
- LINE WORKS: 450円 → TeamChatスタンダード500円はやや高いが機能で上回る
- Teams: 599円（但し365バンドルで実質0円）→ 価格では勝てない
- Chatwork: 700円 → TeamChat500円で価格優位
- Slack: 925円 → TeamChat500円で大幅に安価

■ ターゲット市場

第1ターゲット（最優先）:
- 従業員50〜500人の中堅企業
- 製造業、金融、公共セクターなどセキュリティ要件の厳しい業種
- 現在メール+電話中心でチャット未導入の企業（導入率が低い300-999人層が急成長中）

第2ターゲット:
- Slackの高コストに不満を持つスタートアップ・テック企業
- Teams疲れを感じている大企業内の個別部署
- 自治体・教育機関（ISMAP対応が差別化要因）

■ チーム

- 創業者/CEO: 元Slack日本法人のエンジニアリングマネージャー（Slack社内のUX課題を熟知）
- CTO: 元LINE WORKSの開発リード（現場向けUI設計に精通）
- VP of Sales: 元Chatworkの営業部長（中小企業向け営業に精通）
- メンバー: 12名（エンジニア8名、セールス2名、CS2名）
- 設立: 2025年6月
- 資金調達: シード1.5億円（国内VC、Coral Capital）

■ 現在の状況と実績

ベータ版:
- 2026年1月にクローズドベータ開始
- 参加企業: 30社（製造業12社、IT企業8社、金融3社、公共2社、その他5社）
- 月間アクティブユーザー: 約800人
- NPS: +45（SaaS平均の+30を大幅に上回る）
- 有料転換率: 40%（ベータ参加企業のうちスタンダード以上を契約）
- 平均DAU/MAU比率: 68%（高いエンゲージメント）

ユーザーフィードバック（ベータ）:

- 「AI議事録が想像以上に便利」（製造業 営業部長）
- 「オンプレミスで検討できるのはこれだけ」（自治体 情報政策課）
- 改善要望: 「連携サービスをもっと増やしてほしい」「モバイルアプリの動作を軽くしてほしい」
