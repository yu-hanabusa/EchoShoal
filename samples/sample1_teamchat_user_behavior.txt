日本ビジネスチャットユーザー行動・ニーズレポート
出典一覧は末尾に記載

■ ユーザーの利用実態（日経BPコンサルティング 2024年10月調査、3,300人対象）

利用頻度:
- ほぼ毎日、頻繁に活用: 63.4%
- 時々活用: 23.5%
- 合計で8割超が日常的に利用

業務への貢献実感:
- とても役に立っている: 54.3%
- やや役に立っている: 36.1%
- あまり役に立っていない: 7.5%
- まったく役に立っていない: 2.1%

■ ユーザーが求める機能・改善要望

1. 検索機能の強化: 34.0%（最も多い要望）
2. スケジュール・他ツール連携: 25.8%
3. 既読・未読のステータス管理: 24.2%
4. セキュリティ機能の強化: 14.1%
5. メンション・通知の改善: 10.5%
6. 情報の整理・分類機能（自由回答で多数指摘）

■ ツール選定時の決め手（導入企業の選定基準）

1. 使いやすさ: 21.6%
2. セキュリティ: 20.6%
3. 業務効率化への貢献: 15.6%
4. 既存ツールとの連携
5. 価格・コスト
6. 国産・日本語サポート: 特に中小企業で重要視

■ 導入理由（なぜビジネスチャットを導入したか）

1. スピーディーなコミュニケーション: 23.6%
2. 会議時間の短縮: 15.7%
3. 複数人での情報共有の容易化: 13.9%

■ 業種別ユーザーペインポイント

【製造業】
- 現場（工場）と本社間のコミュニケーション断絶が最大の課題
- 現場作業員はPC非保有が多く、スマホ対応が必須
- 2025年Q1のセキュリティインシデントで「製造業」が業種別最多 → セキュリティ意識が急上昇
- サプライチェーン攻撃が増加しており、外部ツール導入に慎重
- 導入決定者: 工場長または本社IT部門（現場提案→本社承認の二段階）

【金融業】
- 金融庁ガイドラインでクラウドサービスにSOC2等の第三者監査報告書が求められる
- 個人情報・取引情報の取り扱いに厳格なルールあり
- 海外サーバーへのデータ保管に対する強い抵抗感
- 導入検討期間: 通常6ヶ月〜1年以上（稟議・監査・セキュリティ審査）
- 「サイバー対処能力強化法」（2025年5月公布）で金融機関のIT管理が一層厳格化
- 導入決定者: 情報システム部門長 → CISO → 取締役会承認

【公共セクター（自治体・教育機関）】
- ISMAP登録が事実上の必須要件
- 年度予算制のため導入時期が年度初めに集中
- 入札・プロポーザル方式での調達が多い
- 住民情報・学生情報の保護が最優先
- 導入決定者: 情報政策課 → 財政課（予算承認）→ 議会承認（大型案件）

【IT企業・スタートアップ】
- すでにSlackを利用している企業が多い
- 連携エコシステム（GitHub, Jira, Notion等）への依存度が高い
- エンジニアの意見が導入決定に強く影響
- 導入決定者: エンジニアリングマネージャーまたはCTO

■ ユーザーペルソナ（想定される利用者像）

【ペルソナ1: 中堅製造業の情シス担当 田村さん（45歳）】
- 従業員300人の製造業で情報システム部門に1人
- メールとFAXが主要コミュニケーション手段
- 経営層から「DXを進めろ」と言われているが何から始めるか分からない
- セキュリティは気にするが、自分で判断する知識が不足
- 予算は月額15万円程度（300人×500円）なら稟議を通せる
- 決め手: 「国産で日本語サポートがしっかりしている」「導入実績がある」

【ペルソナ2: 地方銀行のIT部門長 佐々木さん（52歳）】
- 従業員500人の地方銀行のIT部門管理者
- 現在Teamsを使用しているが、現場（支店）での利用率が低い
- 金融庁の監査対応に時間を取られる
- オンプレミスで自行サーバーに置きたい
- 予算は高くても問題ないが、稟議に1年かかる
- 決め手: 「ISMAP対応」「オンプレミス」「SOC2取得済み」

【ペルソナ3: スタートアップのCTO 山本さん（32歳）】
- 30人のSaaSスタートアップ
- Slackを使っているがコストが気になり始めた（月額27,750円 = 30×925円）
- GitHub, Notion, Linearとの連携が仕事の生命線
- 新しいツールを試すことに抵抗はない
- 決め手: 「Slackと同等の連携」「コスト半額」「API品質」

【ペルソナ4: 小売チェーンの店長 鈴木さん（38歳）】
- 50店舗のチェーン、各店舗5-10人のスタッフ
- LINE WORKSを使っているが、本部との連携に限界を感じている
- スタッフのITリテラシーは低い（LINEは使える）
- 月額コストは最小限に抑えたい
- 決め手: 「LINEと同じくらい簡単」「無料プランがある」

【ペルソナ5: 何も使っていない中小企業の社長 高橋さん（58歳）】
- 従業員80人の建設会社
- メール、電話、対面のみでコミュニケーション
- チャットツールの存在は知っているが「うちには必要ない」と考えている
- 若手社員から「チャットを入れてほしい」と言われている
- 決め手: 「使わなくても困らないから」（非導入の最大理由）

■ 各ツールのユーザー層の特徴と乗り換え動機

【Microsoft Teamsユーザー】
- 大企業中心（上場企業の事実上の標準）
- Office 365バンドルで追加コストなし
- 「Teams疲れ」が社会現象化（通知過多・UIの複雑さ）
- 乗り換え動機: UIの簡便さへの渇望
- 推定ユーザー数: 国内企業の65%にインストール済み

【Slackユーザー】
- テック企業・スタートアップ中心
- 2,600+の連携サービスに依存（ロックイン効果）
- 月額925円は競合の約2倍で最も高額
- 乗り換え動機: コスト削減
- 乗り換え障壁: 豊富な連携サービスへの依存
- 国内20万社、3万社以上が有料

【LINE WORKSユーザー】
- 小売・建設・自治体の現場系中心
- LINEと同じUIで「導入教育がいらない」と高評価
- 46万社以上、約500万ユーザー
- 乗り換え障壁: LINEに慣れたユーザーの再教育コスト

【Chatworkユーザー】
- 中小企業中心、導入97万社（国内最多）
- 無料プランが入口として機能
- サポート品質とガイドの分かりやすさが高評価
- 乗り換え障壁: 社内の利用習慣の変更

【未導入層（全体の48.9%）】
- 99人以下の企業では78.4%が未導入
- 「必要性を感じない」が最大の非導入理由
- 導入のきっかけ: トップダウン（経営者指示）またはボトムアップ（若手提案）

■ スイッチングコスト（乗り換えの障壁）

技術的コスト:
- データ移行（メッセージ履歴、ファイル）: 数週間〜数ヶ月
- API連携の再構築（Slackは2,600+、Teamsは2,000+連携あり）
- SSO/ID管理の切り替え
- CloudFuze/Cloudiwayなどの移行ツールが存在するが完全ではない

組織的コスト:
- 全従業員への再教育・トレーニング: 1〜3ヶ月
- ワークフロー・運用ルールの再設計
- 移行期間中の二重運用の負担（旧ツールと新ツールの並行利用）
- IT部門の工数（中小企業では「一人情シス」が多く余裕がない）

心理的コスト:
- 慣れたツールからの変更への抵抗感
- 「今のままで困っていない」というイナーシャ
- 導入担当者の責任リスク（失敗した場合の説明責任）
- 「みんなが使っているから」のバンドワゴン効果（Teams/Slack）

■ 導入の意思決定プロセス（典型的な日本企業）

中小企業（50-300人）:
1. 若手社員またはIT担当が提案（1-2ヶ月）
2. 複数ツール比較・トライアル（1-2ヶ月）
3. 決裁（社長または役員）（2週間〜1ヶ月）
4. 全社展開（1-2ヶ月）
合計: 3-7ヶ月

中堅企業（300-1000人）:
1. IT部門が要件定義（1-2ヶ月）
2. ベンダー選定・RFP（2-3ヶ月）
3. セキュリティ審査（1-2ヶ月）
4. 経営会議承認（1ヶ月）
5. 段階的展開（3-6ヶ月）
合計: 8-14ヶ月

金融・公共:
1. 要件定義（2-3ヶ月）
2. 入札/プロポーザル（2-3ヶ月）
3. セキュリティ監査・ISMAP確認（3-6ヶ月）
4. 稟議・承認（2-3ヶ月）
5. 導入・教育（3-6ヶ月）
合計: 12-21ヶ月

■ 料金比較（月額/ユーザー、有料最安プラン基準）

- Microsoft Teams: 599円（Business Basic）※Office 365バンドルなら実質0円
- Chatwork: 700円（ビジネスプラン）
- Slack: 925円（プロプラン）※最も高額
- LINE WORKS: 450円（スタンダード）
- Google Chat: Google Workspaceバンドル（680円～）
- TeamChat: 500円（スタンダード）

■ テレワーク動向（ビジネスチャット需要の背景）

テレワーク実施率:
- 2025年7月: 22.5%（前年同期比横ばい、定着傾向）
- 2025年3月: 17.0%（前年同月と同じ）
- 政府目標: 25.0%（2025年度）

テレワーク実施頻度:
- 週1日未満: 29.1%
- 週1日程度: 20.3%
- 週1日以下の合計: 49.4%（前年43.6%から増加 → 頻度は減少傾向）

企業の出社回帰:
- リモート未実施/予定なし企業: 22.6%（増加傾向）
- 全社員の50-80%がリモート実施: 32.3%（減少傾向）

従業員の希望:
- テレワークを「続けたい」: 82.2%（過去最高）
- 企業の出社回帰方針と従業員の希望にギャップあり

■ フリーミアムモデルの転換率（SaaS全般の参考値）

- 無料トライアル → 有料版転換率: 数%～25%程度（製品・顧客により大幅に異なる）
- Chatwork: 97万社利用のうち有料プランの比率は非公開
- TeamChatのベータ版転換率40%はSaaS平均を大幅に上回る水準

■ セキュリティインシデント動向（導入判断への影響）

- 2025年Q1のセキュリティインシデント: 111件（業種別で「製造」が最多）
- サプライチェーン攻撃が増加: 子会社・委託先経由の侵入が相次ぐ
- 「サイバー対処能力強化法」（2025年5月公布）: 基幹インフラ事業者のIT管理義務強化
- これらの背景から「国産」「国内DC」「セキュリティ認証」が選定基準として上昇傾向
