## 4. 文書処理フロー

```
入力: PDF/DOCX/TXT ファイル or GitHub URL
  │
  ├── GitHub URL の場合
  │   └── fetcher.py: GitHub API → README.md テキスト取得
  │
  └── ファイルの場合
      └── parser.py: ファイル → プレーンテキスト抽出
                │
                ▼
        ┌───────────────────────────────────────┐
        │   ルールベース辞書 + LLM 解析           │
        │                                        │
        │  テキスト                               │
        │    │                                   │
        │    ├─ ルールベース辞書（正規表現）        │
        │    │  ├─ 技術パターン (Python, AWS等)    │
        │    │  └─ 政策キーワード (DX推進法等)     │
        │    │                                   │
        │    └─ LLM抽出                           │
        │       ├─ 組織名・サービス名を抽出        │
        │       └─ 失敗時: 既知名の正規表現FB     │
        │                                        │
        │  出力: AnalysisResult                  │
        │  ├─ entities: [名前, タイプ, 出現位置]  │
        │  ├─ organizations: [組織名] (LLM)      │
        │  ├─ technologies: [技術名] (辞書)      │
        │  └─ policies: [政策名] (辞書)          │
        └──────────────────┬────────────────────┘
                           │
                           ▼
        ┌───────────────────────────────────────┐
        │        Neo4j格納                       │
        │                                        │
        │  Document ノード作成                    │
        │    │                                   │
        │    ├── MENTIONS → Skill ノード          │
        │    ├── MENTIONS → Company ノード        │
        │    └── MENTIONS → Policy ノード         │
        │                                        │
        │  既存ノードがあればリンク                 │
        │  なければ新規作成                        │
        └───────────────────────────────────────┘
```

## 6. レポート生成フロー

```
入力: シミュレーション結果 (全RoundResult)
  │
  ├── DataExtractor
  │   ├─ 全ラウンドのディメンション推移を抽出
  │   ├─ マクロ指標の推移を抽出
  │   ├─ エージェント別アクション集計
  │   ├─ イベント一覧
  │   └─ 文書参照ログ
  │
  ▼
┌────────────────────────────────────────────────┐
│          Claude API / Heavy LLM Provider        │
│                                                  │
│  1. 成功スコア算出                               │
│     ├─ 入力: 最終ディメンション値 + マクロ + 集計 │
│     ├─ 出力: score (0-100), verdict,             │
│     │        key_factors, risks, opportunities   │
│     └─ 判定: 70+ = 成功見込み                    │
│              40-69 = 要注意                      │
│              0-39 = 困難                         │
│                                                  │
│  2. エグゼクティブサマリー                        │
│     └─ 3-5文の総合評価                           │
│                                                  │
│  3. 市場インパクト分析                            │
│     └─ マクロ指標の推移とその意味                  │
│                                                  │
│  4. ディメンション分析                            │
│     └─ 8次元それぞれのトレンドナラティブ           │
│                                                  │
│  5. ステークホルダー影響分析                       │
│     └─ 各ステークホルダータイプへの影響            │
│                                                  │
│  6. 資料影響分析                                  │
│     └─ 入力文書がエージェント判断にどう影響したか   │
│                                                  │
│  7. 追加情報提案                                  │
│     └─ 精度向上に必要な追加データの提案            │
│                                                  │
│  8. 提言                                         │
│     └─ サービス提供者/投資家/ユーザー企業向け      │
└──────────────────────┬─────────────────────────┘
                       │
                       ▼
              SimulationReport (JSON)
              ├─ title
              ├─ success_score
              ├─ executive_summary
              ├─ sections[] (6セクション)
              └─ generated_at
```

## 10. 知識グラフ（Neo4j）データ構造

```
┌─────────────────────── Neo4j Graph Structure ──────────────────────┐
│                                                                     │
│  ┌─────────┐     STATE_AT      ┌───────────────┐                   │
│  │  Agent  │ ──────────────── ▶│ AgentSnapshot │                   │
│  │         │    {round: N}     │  revenue       │                   │
│  │ agent_id│                   │  headcount     │                   │
│  │ name    │                   │  reputation    │                   │
│  │ type    │                   │  satisfaction  │                   │
│  └────┬────┘                   └───────────────┘                   │
│       │                                                             │
│       │ PERFORMED                                                   │
│       │ {round: N}                                                  │
│       ▼                                                             │
│  ┌──────────────┐    CAUSED    ┌──────────────┐                    │
│  │ ActionRecord │ ───────── ▶ │ MarketEffect │                    │
│  │              │              │              │                    │
│  │ action_type  │              │ demand_delta │                    │
│  │ description  │              │ supply_delta │                    │
│  │ visibility   │              └──────┬───────┘                    │
│  │ round        │                     │                            │
│  └──────────────┘                     │ AFFECTS                    │
│                                       ▼                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │  Skill   │  │ Company  │  │  Policy  │  │ Document │          │
│  │  (技術)  │  │  (企業)  │  │  (政策)  │  │  (文書)  │          │
│  └────┬─────┘  └────┬─────┘  └──────────┘  └────┬─────┘          │
│       │             │                            │                 │
│       │ SKILLED_IN  │                  MENTIONS   │                 │
│       │             │              ┌─────────────┘                 │
│       ▼             │              ▼                               │
│  ┌──────────┐       │         Document → MENTIONS → Skill          │
│  │  Agent   │       │         Document → MENTIONS → Company        │
│  │ (技術力) │       │         Document → MENTIONS → Policy         │
│  └──────────┘       │                                              │
│                     │                                              │
│  因果チェーン:                                                      │
│  Agent → PERFORMED → ActionRecord → CAUSED → MarketEffect         │
│                       → AFFECTS → Skill/Dimension                  │
│                                                                     │
│  可視性制御:                                                        │
│  ActionRecord.visibility = public | partial | private              │
│  GraphRAGRetriever が観察者ごとにフィルタリング                      │
└─────────────────────────────────────────────────────────────────────┘
```
